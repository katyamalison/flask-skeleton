Overview

Initially, my goal was to compare the lyrical content of the bands Pavement and Smashing Pumpkins, to solve their decades-old beef and determine which band was the better band. I realized that it would be misrepresentative to use only lyrics from the two bands- context is important. Unfortunately, there are no readily accessible datasets of song lyrics on the internet that could be used for my goals. The closest thing was a bag-of-words dataset that was really outdated and contained no information of genre or release year. Thus, I had to compile the data myself, and doing so led me to a broader, more interesting project.

The data

Compiled using a variety of sources:

  • Billboard API
  • Genius lyrics API + web scraping
  • Lastfm API
  • Bing API for images

I started by getting the Billboard Hot 100 chart from May 12th of every year between 1970 and 2017, and making a set of the artists on those charts. Using the artist names, I was able to find a list of songs by said artist using the Genius API, and scraped the lyrics from the webpage. I used the Lastfm api to get genre tags for each artist. This shit takes forever to run, but the resulting database:

  • Over 1,000 of the most popular artists from the past ~50 years
  • Top 25 most popular songs per artist
  • Information on genre and release year

Looking at the lyrics

Lyrics can be vital to a listener's overall impression of a song, but I daresay the actual *sound* of a song is more important. The characteristics I ultimately sought out were based on what I look for in lyrics.

  • Point of view
  • Alliteration
  • Topically unique (song)
  • Topically diverse (artist)
  • Repetitiveness

Improvements

Some characteristics I would like to add to the analysis:

  • Narrative
  • Metaphor detection
  • Cliche

Additionally, I think there should be another view so users can explore the data, not just compare two artists.
